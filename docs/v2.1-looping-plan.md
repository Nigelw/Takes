# Takes v2.1 — Looping

Focus of the release: **looping**. Two features, the second building on the first.

1. **Repeat Mode** — Off / One / Switch & Repeat, exposed in the Playback menu and a toolbar button.
2. **Loop Selection** — drag a region in the waveform to loop a subsection across tracks.

---

## 1. Repeat Mode

### Behaviour

Three modes:

- **Off** — reaching the end of the playable range stops playback, leaving the playhead parked at the end. The next Play rewinds to the start (of the track, or the loop when one is active).
- **Repeat One** — reaching the end restarts the *same* track from the beginning of the timeline.
- **Switch & Repeat** — reaching the end switches to the *next* track (wrapping) and restarts from the beginning.

"The end" means the timeline end normally, or the loop end when a loop is active (see §2).

With a single track, Switch & Repeat behaves like Repeat One (there is no next track).

### Surfaces

- **Playback menu → Repeat** submenu with Off / One / Switch, current mode checkmarked (a `Picker` bound to `repeatMode`).
- **Toolbar button** in `transportBar`, positioned **between the time readout and the zoom control**. Clicking cycles Off → One → Switch → Off.
  - Icon: `repeat` (Off, secondary/dimmed), `repeat.1` (One, accent), `repeat` (Switch, accent). Off vs Switch share a glyph and are distinguished by tint. Tooltip names the current mode.

---

## 2. Loop Selection

Builds on Repeat Mode to loop a subsection of the timeline.

### Behaviour

- **Select** by dragging horizontally within the waveform area. A drag (movement past a small threshold) creates a selection; a plain click still seeks/moves the playhead. **Shift-click** selects the range between the current playhead and the clicked point.
- **Visual indication**: a translucent rectangle spanning all lanes between the loop start and end, with brighter vertical edges.
- Loop becomes **active on mouse-up**.
- On activation, the **playhead jumps to the loop start**.
- Loop is **resizable** by dragging either edge (start or end). Edges are clamped so `start < end`, a minimum length is enforced, and both stay within the timeline.
- **Deselect**:
  - Edit menu → **Deselect** (⌘D), disabled when there is no loop.
  - **Clicking outside** the selected area clears the loop **and** seeks the playhead to the click point (per product decision).
- When a loop is active, **only that portion of audio plays** — playback wraps at the loop end.
- Activating a loop **auto-switches repeat mode to Switch & Repeat** (so the same loop is heard across tracks). **Deselecting a loop always turns repeat off.**

### Interaction decisions (confirmed with product)

- **Drag = select, click = seek.** The current drag-to-scrub behaviour on the waveform is replaced.
- **Click outside the loop = deselect + seek** in one gesture.
- **Deselect turns repeat off.**
- **Resize handles** are drawn distinctly from the playhead (a white, blue-outlined capsule vs. the thin solid-blue playhead) and stay draggable even when the playhead sits directly on top of them (the playhead does not hit-test).

## Implementation

### Data model — `Models.swift`

```swift
enum RepeatMode: String, CaseIterable {
    case off, one, switchAndRepeat

    var next: RepeatMode { … }        // cycle order for the toolbar button
}

struct LoopRegion: Equatable {
    var start: TimeInterval
    var end: TimeInterval             // invariant: start < end

    static let minimumLength: TimeInterval = 0.05

    /// Clamp to the timeline, enforce start < end and minimum length. Returns nil
    /// if the timeline is too short to hold a loop.
    static func normalized(start:, end:, timelineStart:, timelineEnd:) -> LoopRegion?
}
```

Add to `ComparisonSession`:

```swift
var repeatMode: RepeatMode = .off
var loopRegion: LoopRegion?

var playbackStart: TimeInterval { loopRegion?.start ?? timelineStart }
var playbackEnd:   TimeInterval { loopRegion?.end   ?? timelineEnd }
```

### Playback logic — `PlaybackController.swift`

- **End-of-range handling** in `refreshTransportTick`: change the `transport >= timelineEnd` check to `transport >= session.playbackEnd`, then branch on an extracted pure helper:

  ```swift
  nonisolated static func advanceAtEnd(mode: RepeatMode, canSwitch: Bool) -> EndAction
  // .stop | .restart | .switchThenRestart
  ```

  - `.stop` → stop and rewind the playhead to `playbackStart`.
  - `.restart` → `restartPlayback(from: session.playbackStart)`.
  - `.switchThenRestart` → `selectNextTrack()` then `restartPlayback(from: session.playbackStart)`.

- `restartPlayback(from:)` — reschedules players at the given global time and keeps playing (reuses `reschedulePlayers` + `startScheduledPlayers` + resets `playbackStartedAt/From`). No stop/scroll teardown.
- **Bound scheduled audio to the loop end** in `scheduleTrack`: when a loop is active, cap `framesRemaining` at the loop-end frame so audio never plays past the loop. Worst case is a ≤ one-tick (~50 ms) silent tail before the wrap; nothing beyond the loop is ever heard. (Sample-accurate seams via completion handlers are a possible later refinement.)
- **Seek clamping**: while a loop is active, clamp seeks to `[playbackStart, playbackEnd]` so skip/seek stay inside the loop. (Deselect-and-seek clears the loop first, so it isn't clamped.)
- **Public API**:
  - `setRepeatMode(_:)`, `cycleRepeatMode()`
  - `beginLoop(_ region: LoopRegion)` — set `switchAndRepeat`, set region, seek to `region.start` (reschedule if playing).
  - `resizeLoop(start:?, end:?)` — normalized update of one edge.
  - `deselectLoop()` — clear region and turn repeat off.
- **Housekeeping**: in `recalculateSessionDuration`, clamp or drop `loopRegion` if it falls outside the new timeline; `clearTracks` clears the loop and restores repeat mode.

### UI — `ContentView.swift`

- **Toolbar**: insert a repeat button in `transportBar` between the readout `Text` and `zoomControls`.
- **Loop selection gesture & visuals** live in the `trackTimelineSection` ZStack (same layer as the playhead, so they span all lanes at `trackTimelineHeight`):
  - Remove the per-lane `DragGesture` seek from `waveformLane`.
  - Add one interaction layer over the waveform column (x ∈ `[trackInfoWidth, trackInfoWidth+waveformWidth]`) with a `DragGesture(minimumDistance: 0)`:
    - `onChanged`: once movement exceeds a small threshold, maintain a `@State loopDraft` (start/current global time) and draw it; below threshold, treat as a potential click.
    - `onEnded`: drag → commit via `beginLoop`; click → seek, and if a loop exists and the click is outside it, `deselectLoop()` then seek.
  - Draw the **selection rectangle** (translucent fill + edge lines) between the loop/draft start and end x, full lane-stack height.
  - Two **resize handles** (thin grab strips at the start/end x, above the interaction layer so they take gesture priority) each with a `DragGesture` calling `resizeLoop`.
- Map x ↔ global time with the existing `globalTime(atX:width:)` / `xPosition(for:width:)`.

### Menus — `TakesApp.swift`

- **Playback menu**: add a `Menu("Repeat")` containing a `Picker` bound to `controller.session.repeatMode` (checkmarked Off/One/Switch).
- **Edit menu**: add **Deselect** (⌘D) via `CommandGroup(after: .pasteboard)` (fall back to `CommandMenu("Edit")` if no Edit menu is present), calling `controller.deselectLoop()`, disabled when `loopRegion == nil`.

### Tests — `Tests/TakesTests/`

Cover the extracted pure logic (Swift Testing, matching existing files):

- `RepeatMode.next` cycle order.
- `advanceAtEnd(mode:canSwitch:)` for all mode/track-count combinations.
- `LoopRegion.normalized` — clamping, min-length, start<end, timeline-too-short → nil.
- `ComparisonSession.playbackStart/End` with and without a loop.
- Deselect restores the prior repeat mode; begin-loop forces Switch & Repeat and seeks to start.
