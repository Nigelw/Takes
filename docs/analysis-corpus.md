# Analysis Corpus Manifest

Benchmark test corpus for the experimental audio-analysis feature (see
`docs/experimental-audio-analysis.md`). Generated by
`scripts/make-analysis-corpus.sh` (idempotent, re-run any time) into:

```
/Users/Nigel/Developer/Takes/Private/Analysis Corpus/
```

`Private/` is gitignored and absent from worktrees, so the corpus itself is
never committed -- only the generator script and this manifest are. Re-running
the script with no arguments regenerates anything missing; pass `--force` to
regenerate everything, or a path argument to write elsewhere.

Spectrogram PNGs for every file live in `_spectrograms/` (same basename).
Raw verification numbers (re-measured on every run) live in
`_verification.tsv` in the corpus dir.

All measurements below were produced by the script's verification pass:
- Integrated LUFS / true peak: `ffmpeg -af loudnorm=print_format=summary` first-pass ("Input") measurement.
- HF RMS: `ffmpeg -af "highpass=f=17000,astats=measure_overall=RMS_level"`, RMS in dBFS of everything above 17 kHz.
- Empirical cutoff points below were additionally confirmed with a 14-20 kHz highpass sweep and by eye on the spectrograms.

## Sources

### `reference.wav` (synthetic, primary)

30 s / 44.1 kHz / stereo / 24-bit. Built entirely in `ffmpeg` filtergraphs
(`anoisesrc`, `aevalsrc`, `amix`, `alimiter`), fully reproducible, no external
input. Contents, mixed together:

- Two independent pink-noise beds (L/R, different seeds so the stereo image is decorrelated), amplitude 0.12.
- Two harmonic-stack tones standing in for "melodic" content (summed sine partials at 110/220/330/440 Hz and 164.81/329.63/494.44 Hz).
- A percussive burst train: white noise gated by `exp(-35 * (t mod 1.7))`, i.e. a fast-decay hit repeating every 1.7 s.
- A slow linear chirp sweep from 200 Hz to 21 kHz across the full 30 s (`aevalsrc` with the proper quadratic phase term `sin(2*pi*(f0*t + k/2*t^2))`, k=(f1-f0)/duration) -- this gives the corpus genuine full-bandwidth content right up to the cutoff-detector's operating range.
- A ~2.5 s near-silence gap at t = 19.0-21.5 s (gain dropped to 0.02x) for noise-floor ground truth.
- Everything summed then passed through `alimiter` with a linear ceiling of 0.708 (-3 dBFS).

Ground truth: true peak measured at -2.1 dBTP (target was -3 dBFS; close enough -- the limiter's soft knee left a little headroom). Integrated loudness is about -13.6 LUFS. Full energy present to ~21 kHz (no encoder has ever touched this signal).

### `real_reference.wav` (real music, secondary)

30 s window (t = 20-50 s, skipping the intro) decoded from
`Private/Audio Samples/11 Where to Begin.m4a`, resampled/remuxed to
44.1 kHz / stereo / 24-bit PCM. This source is already AAC-sourced -- the
original file is a ~283 kbps AAC encode (confirmed via `afinfo`), so it carries
whatever high-frequency character that encoder left behind (soft rolloff
approaching ~19-20 kHz is visible on its spectrogram, not a hard shelf).
Treat `real_reference.wav` as "known-good-ish, not textbook-lossless" -- it's
here for realism (real transients, real mix, real stereo image), not as a
pristine ground truth like the synthetic reference. Measured: -12.3 LUFS,
true peak -0.0 dBTP.

## Phase 2: provenance-detection cases

Nine additional files targeting analog-source detection (hiss/clicks/crackle/
rumble, and whether hiss is stereo-decorrelated or a mono duplicate) and
lossy-encode-quality contrasts (encoder-era LAME-tag presence, intensity-
stereo joint coding, and pre-echo around sharp transients). Generated by the
same script; see `build_real_vinyl_sim`, `build_real_tape_sim`,
`build_mp3_192_modern/early/intensity`, and `build_transient_reference` /
`build_transient_variants` in `scripts/make-analysis-corpus.sh`. Small Python
helper scripts (`gen_clicks.py`, `gen_crackle.py`, `gen_transients.py`) are
written by the shell script into `_gen/` inside the corpus dir on every run
(not committed -- regenerated, kept only for inspectability).

A second verification pass (`verify_phase2`) writes
`_verification_phase2.tsv` alongside the phase-1 `_verification.tsv`, with
decorrelation, side-channel rumble, tag-offset, and cutoff-sweep numbers used
in the tables below.

### Analog-source simulations (gapless)

Applied to `real_reference.wav`, which has no quiet passages -- deliberately,
since a detector that only finds hiss/rumble in a silent window would miss it
entirely on a real recording. Both files add their noise across the *entire*
30 s, no gaps.

| File | Recipe | Ground truth |
| --- | --- | --- |
| `real_tape_sim.wav` | Stereo-**decorrelated** hiss only: two independently seeded `anoisesrc=white` sources (seeds 611/612), one per channel, mixed in at a calibrated gain to hit ≈ -50 dBFS overall RMS. No clicks/crackle/rumble. | Constant, gapless hiss floor whose L and R components are statistically independent -- the "real tape" contrast case for a stereo-coherence detector. |
| `real_vinyl_sim.wav` | All four vinyl artifacts combined, gapless, on `real_reference.wav`: (a) stereo-decorrelated hiss (seeds 511/512, independent `anoisesrc=white`, calibrated to ≈ -55 dBFS overall); (b) clicks: a precomputed sparse impulse WAV from `gen_clicks.py` (irregular timing, min 300 ms apart, 0.5-2 ms wideband spikes with a fast exponential-decay envelope, amplitude 0.5-0.95, mixed in equally to both channels); (c) crackle: dense low-level impulsive noise from `gen_crackle.py` (0.1-0.6 ms micro-impulses, ~120/s, amplitudes weighted toward quiet with occasional louder pops, peak amp 0.15); (d) rumble: brown noise cascaded through two 30 Hz lowpass biquads, calibrated to -35 dBFS RMS pre-upmix, then panned to the SIDE channel only (L=+n, R=-n) via `pan=stereo\|c0=c0\|c1=-1*c0`. | Gapless, multi-artifact vinyl-rip simulation. Click count is randomized per run (seeded, so reproducible) -- see measured count below. Rumble is present *only* in the side channel by construction; the source's own bass content still dominates the mid channel in absolute terms (see note below). |

**Is `real_hiss.wav` (v1) correlated or decorrelated?** Checked directly:
`real_hiss.wav` and `hiss.wav` both add a *single* `anoisesrc` source
duplicated to both channels via `aformat=channel_layouts=stereo` before
mixing -- i.e. **fully correlated (mono) hiss**, not stereo noise. This makes
them the correct contrast case for `real_tape_sim.wav`/`real_vinyl_sim.wav`'s
decorrelated hiss: same general "add broadband noise" recipe, opposite
stereo-coherence ground truth. Confirmed empirically below.

Notes on the numbers:
- `true_lossless.flac` matches `reference.wav` exactly, as expected for a non-lossy container.
- `fake_lossless_mp3128.flac` matches `mp3_128.mp3` exactly (same LUFS, same HF RMS) -- the FLAC wrapper doesn't change the underlying (already-lossy) signal, which is exactly the property the "is this really lossless" detector needs to catch.
- `tilt_bassy.wav`'s HF RMS (-26.8) is indistinguishable from `reference.wav` (-26.8) -- correct, since a 120 Hz low shelf shouldn't move energy above 17 kHz.
- `tilt_bright.wav`'s HF RMS (-20.8) is about 6 dB above reference -- correct, matches the +6 dB high-shelf boost at 8 kHz (shelf gain fully applied by 17 kHz).
- `loud.wav`/`quiet.wav` differ by ~16 LU rather than a clean 16 dB (8+8) because `loud.wav` additionally runs through a limiter that reduces crest factor, which nonlinearly affects the gated LUFS measurement -- expected, not a bug.
- Real-music derivatives sit at consistently higher HF RMS values than their synthetic counterparts because the real recording has less energy above 17 kHz to begin with (it's an AAC-sourced track, already rolled off before we touch it) -- the deltas (muffled vs reference, hiss vs reference) are what matter, not the absolute numbers matching the synthetic set.
- `real_tape_sim.wav` (-51.6) and `real_vinyl_sim.wav` (-50.1) land close to their targeted -50/-55 dBFS hiss levels; `real_vinyl_sim.wav` reads slightly louder than `real_tape_sim.wav` in HF RMS because it's also carrying crackle and click energy in the same band, on top of similar hiss.
- `mp3_192_early.mp3` (-38.3) is markedly lower than `mp3_192_modern.mp3`/`mp3_192_intensity.mp3` (both -30.6/-30.7) in HF RMS above 17 kHz -- consistent with its hard 16 kHz lowpass vs. the ~19 kHz-ish rolloff of a normal 192 kbps encode.
- `transient_mp3_128.mp3` and `transient_fake_lossless_mp3128.flac` report identical numbers across the board (LUFS, true peak, HF RMS) -- expected, since the FLAC is a lossless re-wrap of the MP3 and carries the same pre-echo forward untouched.

## Phase 2 measurements: decorrelation, rumble, tags, intensity-stereo

From `_verification_phase2.tsv` (regenerated alongside `_verification.tsv` on
every run). "Side" = RMS of `0.5*(L-R)`, "mid" = RMS of `0.5*(L+R)`, both in
dBFS; smaller mid-side gap (mid closer to side) means more decorrelated/wider
stereo image in that band.

### Hiss coherence: correlated (v1) vs decorrelated (v2), RMS above 10 kHz

| File | Side RMS (dB) | Mid RMS (dB) | Gap (mid - side) |
| --- | --- | --- | --- |
| real_reference.wav (untouched baseline) | -50.0 | -42.8 | 7.2 |
| real_hiss.wav (v1, mono/correlated hiss) | -56.0 | -48.7 | 7.3 |
| real_tape_sim.wav (v2, decorrelated hiss) | -49.0 | -42.6 | 6.4 |
| real_vinyl_sim.wav (v2, decorrelated + clicks/crackle/rumble) | -49.7 | -42.0 | 7.7 |
| hiss.wav (synthetic, v1, mono/correlated) | -47.6 | -29.4 | 17.8 |

At the >10 kHz band, the gap doesn't cleanly separate correlated from
decorrelated on the real-music files -- `real_reference.wav`'s *own* stereo
image already has real (non-hiss) HF content, and at -50/-55 dBFS the added
hiss is comparable in level to that residual, so it doesn't dominate the
measurement. Re-running the same side/mid comparison restricted to >20 kHz,
where `real_reference.wav`'s own AAC-sourced HF content has mostly rolled
off and the added noise dominates, separates them clearly:

| File | Side RMS >20kHz (dB) | Mid RMS >20kHz (dB) | Gap |
| --- | --- | --- | --- |
| real_reference.wav | -67.3 | -63.0 | 4.3 |
| real_hiss.wav (correlated) | -73.3 | -67.8 | 5.6 (tracks the untouched baseline -- mono hiss doesn't add width) |
| real_tape_sim.wav (decorrelated) | -61.8 | -60.1 | 1.7 (collapses toward 0 -- side and mid converge, the decorrelation signature) |
| real_vinyl_sim.wav (decorrelated + clicks/rumble) | -64.9 | -55.7 | 9.2 (noisier -- clicks/crackle/rumble also contribute non-hiss energy in this range) |

Practical implication for the v2 coherence detector: **don't test raw
full-band mid/side ratio and expect a clean signal on real material** --
isolate the noise floor first (e.g. a high-enough band that the source's own
content is negligible, or a spectral-flatness/noise-floor estimate) before
comparing side vs. mid. The synthetic `hiss.wav` case (bottom row) shows what
a clean, unambiguous correlated-noise signature looks like when there's no
competing real-source HF content to confound it.

The purely synthetic decorrelated-noise construction (no real source mixed
in) was independently confirmed during script development to converge to a
~0.0 dB mid-side gap for two independently seeded noise sources and a
~7-18 dB gap for one mono source duplicated to both channels -- the
generation recipe itself is correct; the ambiguity above is specifically a
real-music/at-this-noise-level measurement artifact, not a bug in
`real_tape_sim.wav`/`real_vinyl_sim.wav`.

### Vinyl rumble: side-channel sub-30 Hz level

| File | Side RMS <30Hz (dB) | Mid RMS <30Hz (dB) |
| --- | --- | --- |
| real_reference.wav (baseline, no rumble) | -45.6 | -31.2 |
| real_vinyl_sim.wav (rumble added, side-only) | -35.2 | -31.2 |

The side channel jumps ~10 dB (-45.6 -> -35.2) once rumble is added, while
the mid channel is unchanged (-31.2 both) -- confirming the rumble is
injected purely as a side-channel (L=+n, R=-n) signal, as specified. Note the
mid channel's absolute <30 Hz level (-31.2 dB) is *louder* than the injected
side-channel rumble (-35.2 dB) in both files -- that's the real recording's
own bass/kick content, which the rumble sim doesn't attempt to swamp; the
diagnostic signal is the side-channel *increase*, not an absolute threshold.

Intended/placed click count for the current corpus generation: **16 clicks**
over 30 s (`gen_clicks.py` targets a random rate of 20-40/min per run, seeded

### mp3_192_early vs mp3_192_modern: cutoff sweep and tag presence

| File | >14kHz | >15kHz | >16kHz | >17kHz | >18kHz | >19kHz | >20kHz |
| --- | --- | --- | --- | --- | --- | --- | --- |
| mp3_192_modern.mp3 | -27.3 | -28.1 | -29.2 | -30.6 | -32.8 | -36.5 | -42.8 |
| mp3_192_early.mp3 | -31.7 | -33.4 | -35.6 | -38.3 | -42.0 | -46.9 | -53.9 |

`mp3_192_modern.mp3` keeps meaningful energy out to ~19 kHz before the
typical MP3 near-Nyquist rolloff; `mp3_192_early.mp3` is already 4-11 dB
lower at every point in this sweep and drops off starting well before
16 kHz, confirming its forced `--lowpass 16` cutoff. `mp3_192_intensity.mp3`
tracks `mp3_192_modern.mp3` almost exactly in this sweep (as expected -- the
intensity-stereo simulation changes stereo width, not the overall spectral
envelope).

Tag byte-offset search (`file.find(b"LAME"/"Xing"/"Info")`, -1 = not found):

| File | LAME offset | Xing offset | Info offset | File size |
| --- | --- | --- | --- | --- |
| mp3_192_modern.mp3 | 156 | -1 | 36 | 721,605 B |
| mp3_192_early.mp3 | 720,481 | -1 | -1 | 720,979 B |
| mp3_192_intensity.mp3 | 156 | -1 | 36 | 721,605 B |

`mp3_192_modern.mp3`/`mp3_192_intensity.mp3` both carry a genuine LAME
VBR-tag-frame (LAME writes "Info" for CBR/ABR streams, "Xing" for true VBR --
we're CBR here, so "Info" is correct) right after the first MPEG frame
header at byte 36, with the "LAME3.100..." version string embedded inside
that same tag frame at byte 156. `mp3_192_early.mp3`'s only "LAME" match is
at byte 720,481 -- 498 bytes from EOF, and inspection shows it's a
coincidental match inside a run of `0x55` padding bytes in the last frame,
**not** a structured tag (no "Xing"/"Info" match anywhere in the file, and
the first 4 bytes are a plain `0xFFFB` MPEG frame sync with no tag frame
following). This is the intended contrast: `-t` genuinely suppresses the tag
rather than just moving it.

### Intensity-stereo simulation: decorrelation above/below the 8 kHz split

| File | Side >8kHz (dB) | Mid >8kHz (dB) | Side <8kHz (dB) | Mid <8kHz (dB) |
| --- | --- | --- | --- | --- |
| reference.wav (baseline) | -40.9 | -22.6 | n/a (not measured; below-split behavior is verified via the intensity file itself) | n/a |
| mp3_192_intensity.mp3 | -51.5 | -23.9 | -30.8 | -23.4 |

Above 8 kHz, `mp3_192_intensity.mp3`'s side-channel level drops sharply

mono collapse.

This confirms the spec's key discriminator empirically: `mp3_128`-derived
files (including the fake-lossless FLAC built from it) show a markedly lower
and more sharply-accelerating HF tail than the true-lossless/high-bitrate
files, and the spectrograms show it as a visible hard edge around 16 kHz
(clearest on `mp3_128.png`, where the chirp sweep line visibly terminates as
it crosses ~16 kHz) versus a clean line all the way to ~21 kHz on
`reference.png` and `mp3_320.png`.

## Regenerating

```
scripts/make-analysis-corpus.sh                # generate anything missing
scripts/make-analysis-corpus.sh --force         # regenerate everything
scripts/make-analysis-corpus.sh /some/other/dir # write elsewhere
```

Full regeneration takes well under a minute on Apple Silicon; the verify +
spectrogram pass (which always re-runs) adds roughly 25 s.
